T4TEK Attendance Gateway

Local-first attendance, access control and fingerprint synchronization for ZKTeco devices.

Overview

T4TEK Attendance Gateway connects Odoo with Windows Controllers and ZKTeco attendance/access-control devices. The module uses a desired-state synchronization model so Controllers pull the expected state from Odoo, calculate local deltas, and synchronize only what changed.

Main Features

  • Controller self-registration and token-based authentication.
  • Block / Unblock Controller management from Odoo.
  • Desired-State Sync for Device Users, assignments and deleted users.
  • Fingerprint Master workflow: harvest from source device, upload to Odoo, distribute to multiple devices.
  • Optimized sync_fingerprint_set jobs instead of per-finger command jobs.
  • Attendance log ingestion from local Controllers.
  • OWL fingerprint map widget for 10-finger enrollment visibility.

Operational Workflow

  1. Controller requests /api/entry_control/v1/auth/token.
  2. Controller checks in through /api/entry_control/v1/hello.
  3. Controller uploads harvested fingerprints through /api/entry_control/v1/sync/fingerprints/upload.
  4. Controller pulls desired state through /api/entry_control/v1/sync/manifest.
  5. Controller pulls fingerprint payload only when hashes changed.
  6. Controller synchronizes users and fingerprints to local devices.
